# Linux Basics


Linux is an **open-source operating system** widely used for servers, cloud computing, cybersecurity, software development, and embedded systems. It is known for its stability, security, flexibility, and powerful command-line interface (CLI).

Unlike Windows, Linux is built around the philosophy that **everything is a file**, allowing users to interact with files, directories, devices, and even processes using simple commands.

In this guide, you'll learn:

- What Linux is
- Linux file system structure
- Terminal basics
- Essential Linux commands
- File and directory management
- Permissions
- Processes
- Package management
- Useful shortcuts

> **Note**
>
> Most Linux distributions (Ubuntu, Debian, Fedora, Arch, etc.) share many of the same commands. The examples in this guide primarily use Ubuntu.

---

# What is Linux?

Linux is an operating system kernel created by **Linus Torvalds** in 1991. Today, it powers:

- Web servers
- Cloud platforms (AWS, Azure, Google Cloud)
- Android phones
- Raspberry Pi
- Supercomputers
- Cybersecurity tools like Kali Linux

Some popular Linux distributions include:

- Ubuntu
- Debian
- Fedora
- CentOS
- Arch Linux
- Kali Linux

# Linux File System

Linux uses a single directory tree beginning at the **root directory**.

```text
/
├── bin
├── boot
├── dev
├── etc
├── home
├── lib
├── media
├── opt
├── proc
├── root
├── tmp
├── usr
└── var
```

## Important Directories

| Directory | Purpose |
|------------|----------|
| `/` | Root directory |
| `/home` | User files |
| `/root` | Root user's home |
| `/etc` | Configuration files |
| `/bin` | Essential commands |
| `/usr` | Installed programs |
| `/var` | Logs and variable data |
| `/tmp` | Temporary files |
| `/dev` | Devices |
| `/proc` | Process information |

## Delete

Delete file

```bash
rm file.txt
```

Delete folder

```bash
rm -r Folder
```

Force delete

```bash
rm -rf Folder
```

>  Be careful with `rm -rf`. Deleted files cannot be recovered easily.

# Searching

Search files

```bash
find . -name "*.txt"
```

Search text inside files

```bash
grep "error" app.log
```

Ignore case

```bash
grep -i "error" app.log
```

Recursive search

```bash
grep -r "password" .
```

# File Permissions

View permissions

```bash
ls -l
```

Example:

```text
-rwxr-xr--
```

Breakdown

| Symbol | Meaning |
|---------|----------|
| r | Read |
| w | Write |
| x | Execute |
| - | Permission absent |

Permissions belong to:

- Owner
- Group
- Others

---

## Change Permissions

```bash
chmod 755 script.sh
```

Meaning:

| Number | Permission |
|---------|------------|
| 7 | rwx |
| 6 | rw- |
| 5 | r-x |
| 4 | r-- |

---

## Change Ownership

```bash
sudo chown john file.txt
```
